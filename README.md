## 🧩 Projects (Backend-First, Impact-Oriented)

### 1) Airline Booking Management System (Microservices)
**Problem:** Traditional monolithic booking flows struggle with scale, service isolation, and reliable cross-service coordination.  
**Solution:** Designed a microservices backend with Auth, Booking, Reminder, Flights, and API Gateway; integrated asynchronous communication with RabbitMQ for decoupled workflows.  
**Tech:** Node.js, Express, MySQL, Sequelize, RabbitMQ, API Gateway, JWT  
**Impact / Learning:** Built strong foundations in service boundaries, event-driven communication, auth in distributed systems, and production-like backend architecture.

🔗 https://github.com/KrrishKumar125/Airline-Booking-Management-System

---

### 2) RideWave – Real-Time Ride Matching System
**Problem:** Ride platforms require low-latency matching and continuous location updates at scale.  
**Solution:** Implemented real-time ride matching with Redis geospatial indexing and Socket.IO-based event streams for live driver/rider coordination.  
**Tech:** Node.js, MongoDB, Redis (GeoHashing), Socket.IO  
**Impact / Learning:** Deepened expertise in real-time backend design, pub/sub event handling, and performance-aware location matching systems.

🔗 https://github.com/KrrishKumar125/RideWave-Real-Time-Ride-Matching-System

---

### 3) AlgoForge – LeetCode-like Backend
**Problem:** Online judging systems need secure code execution, asynchronous evaluation, and scalable submission throughput.  
**Solution:** Built a microservices backend with Problem, Submission, and Evaluator services; used queue-driven workflows and Docker-based execution for isolated, scalable evaluation.  
**Tech:** Node.js, Microservices, Queues, Docker, WebSockets  
**Impact / Learning:** Strengthened distributed pipeline design, asynchronous processing patterns, and reliability in compute-heavy backend systems.

🔗 https://github.com/KrrishKumar125/AlgoForge-Backend
